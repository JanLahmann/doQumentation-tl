Palakasin ang pag-uuri ng mga feature gamit ang projected quantum kernels

Tinatayang paggamit: 80 minuto sa isang Heron r3 processor (TANDAAN: Tinatayang halaga lamang ito. Maaaring mag-iba ang inyong runtime.)

Sa tutorial na ito, ipinapakita namin kung paano patakbuhin ang isang projected quantum kernel (PQK) gamit ang Qiskit sa isang tunay na biological dataset, batay sa papel na Enhanced Prediction of CAR T-Cell Cytotoxicity with Quantum-Kernel Methods [1].

Ang PQK ay isang pamamaraan na ginagamit sa quantum machine learning (QML) upang i-encode ang klasikal na data sa isang quantum feature space at i-project ito pabalik sa klasikal na domain, sa pamamagitan ng paggamit ng mga quantum computer upang mapabuti ang pagpili ng feature. Kabilang dito ang pag-encode ng klasikal na data sa mga quantum state gamit ang isang quantum circuit, karaniwang sa pamamagitan ng prosesong tinatawag na feature mapping, kung saan ang data ay binabago sa isang mataas na dimensional na Hilbert space. Ang aspetong "projected" ay tumutukoy sa pagkuha ng klasikal na impormasyon mula sa mga quantum state, sa pamamagitan ng pagsukat ng mga tiyak na observable, upang bumuo ng isang kernel matrix na magagamit sa mga klasikal na kernel-based na algorithm tulad ng support vector machines. Ginagamit ng pamamaraang ito ang mga computational na kalamangan ng mga quantum system upang posibleng makamit ang mas magandang pagganap sa ilang mga gawain kumpara sa mga klasikal na pamamaraan.

Ipinapalagay din ng tutorial na ito ang pangkalahatang pamilyaridad sa mga pamamaraan ng QML. Para sa karagdagang pag-aaral ng QML, sumangguni sa kursong Quantum machine learning sa IBM Quantum Learning.

Mga Kinakailangan​

Bago simulan ang tutorial na ito, tiyakin na naka-install ang mga sumusunod:

  • Qiskit SDK v2.0 o mas bago, na may suporta sa visualization
  • Qiskit Runtime v0.40 o mas bago (pip install qiskit-ibm-runtime)
  • Category encoders 2.8.1 (pip install category-encoders)
  • NumPy 2.3.2 (pip install numpy)
  • Pandas 2.3.2 (pip install pandas)
  • Scikit-learn 1.7.1 (pip install scikit-learn)
  • Tqdm 4.67.1 (pip install tqdm)

Hakbang 1: I-map ang mga klasikal na input sa isang quantum na problema​

Paghahanda ng Dataset​

Sa tutorial na ito, gumagamit tayo ng isang tunay na biological dataset para sa isang binary classification na gawain, na nabuo nina Daniels et al. (2022) at maaaring i-download mula sa supplementary material na kasama ng papel. Ang data ay binubuo ng mga CAR T-cell, na mga genetically engineered T-cell na ginagamit sa immunotherapy upang gamutin ang ilang mga kanser. Ang mga T-cell, isang uri ng immune cell, ay binabago sa laboratoryo upang magpahayag ng mga chimeric antigen receptor (CAR) na nagta-target ng mga tiyak na protina sa mga cancer cell. Ang mga binagong T-cell na ito ay makakakilala at makakasira ng mga cancer cell nang mas epektibo. Ang mga feature ng data ay ang mga CAR T-cell motif, na tumutukoy sa tiyak na istruktura o functional na bahagi ng CAR na naka-engineer sa mga T-cell. Batay sa mga motif na ito, ang aming gawain ay hulaan ang cytotoxicity ng isang ibinigay na CAR T-cell, ina-label ito bilang toxic o non-toxic. Ipinapakita ng sumusunod ang mga helper function upang i-preprocess ang dataset na ito.

    """
    # Read from the csv files

        encoding="unicode_escape",
        sep=",",

    # Fix the last motif ID
    train_data[train_data == 17] = 14
    train_data.columns = [
        "Cell Number",
        "motif",
        "motif.1",
        "motif.2",
        "motif.3",
        "motif.4",
        "Nalm 6 Cytotoxicity",
    ]
    test_data[test_data == 17] = 14
    test_data.columns = [
        "Cell Number",
        "motif",
        "motif.1",
        "motif.2",
        "motif.3",
        "motif.4",
        "Nalm 6 Cytotoxicity",
    ]

    # Adjust motif at the third position
    if args["filter_for_spacer_motif_third_position"]:
        train_data = train_data[
            (train_data["motif.2"] == 14) | (train_data["motif.2"] == 0)
        ]
        test_data = test_data[
            (test_data["motif.2"] == 14) | (test_data["motif.2"] == 0)
        ]

    train_data = train_data[
        args["motifs_to_use"] + [args["label_name"], "Cell Number"]
    ]
    test_data = test_data[
        args["motifs_to_use"] + [args["label_name"], "Cell Number"]
    ]

    # Adjust motif at the last position
    if not args["allow_spacer_motif_last_position"]:
        last_motif = args["motifs_to_use"][len(args["motifs_to_use"]) - 1]
        train_data = train_data[
            (train_data[last_motif] != 14) & (train_data[last_motif] != 0)
        ]
        test_data = test_data[
            (test_data[last_motif] != 14) & (test_data[last_motif] != 0)
        ]

    # Get the labels
    train_labels = np.array(train_data[args["label_name"]])
    test_labels = np.array(test_data[args["label_name"]])

    # For the classification task use the threshold to binarize labels
    train_labels[train_labels > args["label_binarization_threshold"]] = 1
    train_labels[train_labels < 1] = args["min_label_value"]
    test_labels[test_labels > args["label_binarization_threshold"]] = 1
    test_labels[test_labels < 1] = args["min_label_value"]

    # Reduce data to just the motifs of interest
    train_data = train_data[args["motifs_to_use"]]
    test_data = test_data[args["motifs_to_use"]]

    # Get the class and motif counts
    min_class = np.min(np.unique(np.concatenate([train_data, test_data])))
    max_class = np.max(np.unique(np.concatenate([train_data, test_data])))

    num_class = max_class - min_class + 1
    num_motifs = len(args["motifs_to_use"])
    print(str(max_class) + ":" + str(min_class) + ":" + str(num_class))

    train_data = train_data - min_class
    test_data = test_data - min_class

    return (
        train_data,
        test_data,
        train_labels,
        test_labels,
        num_class,
        num_motifs,
    )

def data_encoder(args, train_data, test_data, num_class, num_motifs):
    """
    Use one-hot or binary encoding for classical data representation.
    """
    if args["encoder"] == "one-hot":
        # Transform to one-hot encoding
        train_data = np.eye(num_class)[train_data]
        test_data = np.eye(num_class)[test_data]

        train_data = train_data.reshape(
            train_data.shape[0], train_data.shape[1] * train_data.shape[2]
        )
        test_data = test_data.reshape(
            test_data.shape[0], test_data.shape[1] * test_data.shape[2]
        )

    elif args["encoder"] == "binary":
        # Transform to binary encoding
        encoder = ce.BinaryEncoder()

        base_array = np.unique(np.concatenate([train_data, test_data]))
        base = pd.DataFrame(base_array).astype("category")
        base.columns = ["motif"]
        for motif_name in args["motifs_to_use"][1:]:
            base[motif_name] = base.loc[:, "motif"]
        encoder.fit(base)

        train_data = encoder.transform(train_data.astype("category"))
        test_data = encoder.transform(test_data.astype("category"))

        train_data = np.reshape(
            train_data.values, (train_data.shape[0], num_motifs, -1)
        )
        test_data = np.reshape(
            test_data.values, (test_data.shape[0], num_motifs, -1)
        )

        train_data = train_data.reshape(
            train_data.shape[0], train_data.shape[1] * train_data.shape[2]
        )
        test_data = test_data.reshape(
            test_data.shape[0], test_data.shape[1] * test_data.shape[2]
        )

    else:
        raise ValueError("Invalid encoding type.")

    return train_data, test_data

Hakbang 2: I-optimize ang problema para sa pagpapatakbo sa quantum hardware​

Quantum circuit​

Binubuo na namin ngayon ang feature map na nag-e-embed ng aming klasikal na dataset sa isang feature space na may mas mataas na dimensyon. Para sa pag-embed na ito, ginagamit namin ang ZZFeatureMap mula sa Qiskit.

feature_dimension = train_data.shape[1]
reps = 24
insert_barriers = True
entanglement = "pairwise"

# ZZFeatureMap with linear entanglement and a repetition of 2
embed = ZZFeatureMap(
    feature_dimension=feature_dimension,
    reps=reps,
    entanglement=entanglement,
    insert_barriers=insert_barriers,
    name="ZZFeatureMap",
)
embed.decompose().draw(output="mpl", style="iqp", fold=-1)

Isa pang opsyon sa quantum embedding ay ang 1D-Heisenberg Hamiltonian evolution ansatz. Maaari ninyong laktawan ang pagpapatakbo ng seksyong ito kung nais ninyong magpatuloy gamit ang ZZFeatureMap.

feature_dimension = train_data.shape[1]
num_qubits = feature_dimension + 1
embed2 = QuantumCircuit(num_qubits)
num_trotter_steps = 6
pv_length = feature_dimension * num_trotter_steps
pv = ParameterVector("theta", pv_length)

# Add Haar random single qubit unitary to each qubit as initial state
np.random.seed(42)
seeds_unitary = np.random.randint(0, 100, num_qubits)
for i in range(num_qubits):
    rand_gate = UnitaryGate(random_unitary(2, seed=seeds_unitary[i]))
    embed2.append(rand_gate, [i])

def trotter_circ(feature_dimension, num_trotter_steps):
    num_qubits = feature_dimension + 1
    circ = QuantumCircuit(num_qubits)
    # Even
    for i in range(0, feature_dimension, 2):
        circ.rzz(2 * pv[i] / num_trotter_steps, i, i + 1)
    for i in range(0, feature_dimension, 2):
        circ.rxx(2 * pv[i] / num_trotter_steps, i, i + 1)
    for i in range(0, feature_dimension, 2):
        circ.ryy(2 * pv[i] / num_trotter_steps, i, i + 1)
    # Odd
    for i in range(1, feature_dimension, 2):
        circ.rzz(2 * pv[i] / num_trotter_steps, i, i + 1)
    for i in range(1, feature_dimension, 2):
        circ.rxx(2 * pv[i] / num_trotter_steps, i, i + 1)
    for i in range(1, feature_dimension, 2):
        circ.ryy(2 * pv[i] / num_trotter_steps, i, i + 1)
    return circ

# Hamiltonian evolution ansatz
for step in range(num_trotter_steps):
    circ = trotter_circ(feature_dimension, num_trotter_steps)
    if step % 2 == 0:
        embed2 = embed2.compose(circ)
    else:
        reverse_circ = circ.reverse_ops()
        embed2 = embed2.compose(reverse_circ)

embed2.draw(output="mpl", style="iqp", fold=-1)

Hakbang 3: I-execute gamit ang mga Qiskit primitive​

Sukatin ang mga 1-RDM​

Ang mga pangunahing bahagi ng mga projected quantum kernel ay ang mga reduced density matrix (RDM), na nakukuha sa pamamagitan ng mga projective na pagsukat ng quantum feature map. Sa hakbang na ito, kinukuha natin ang lahat ng single-qubit reduced density matrix (1-RDM), na kalaunan ay ipapasok sa klasikal na exponential kernel function. Tingnan natin kung paano kalkulahin ang 1-RDM para sa isang solong data point mula sa dataset bago natin ulitin ito para sa lahat ng data. Ang mga 1-RDM ay koleksyon ng mga single-qubit na pagsukat ng mga Pauli operator na X, Y at Z sa lahat ng qubit. Ito ay dahil ang isang single-qubit na RDM ay maaaring ganap na ipahayag bilang: ρ=12(I+⟨σ⟩xσx+⟨σ⟩yσy+⟨σ⟩zσz)\rho = \frac{1}{2} \big( I + \braket \sigma_x \sigma_x + \braket \sigma_y \sigma_y + \braket \sigma_z \sigma_z \big)ρ=21​(I+⟨σ⟩x​σx​+⟨σ⟩y​σy​+⟨σ⟩z​σz​) Una, pipili tayo ng backend na gagamitin.

service = QiskitRuntimeService()
backend = service.least_busy(
    operational=True, simulator=False, min_num_qubits=133
)
target = backend.target

Pagkatapos ay pinapatakbo natin ang quantum circuit at sinusukat ang mga projection. Tandaan na binubuksan natin ang error mitigation, kasama ang Zero Noise Extrapolation (ZNE).

# Let's select the ZZFeatureMap embedding for this example
qc = embed
num_qubits = feature_dimension

# Identity operator on all qubits
id = "I" * num_qubits

# Let's select the first training datapoint as an example
parameters = train_data[0]

# Bind parameter to the circuit and simplify it
qc_bound = qc.assign_parameters(parameters)
transpiler = generate_preset_pass_manager(
    optimization_level=3, basis_gates=["u3", "cz"]
)
transpiled_circuit = transpiler.run(qc_bound)

# Transpile for hardware
transpiler = generate_preset_pass_manager(optimization_level=3, target=target)
transpiled_circuit = transpiler.run(transpiled_circuit)

# We group all commuting observables
# These groups are the Pauli X, Y and Z operators on individual qubits
observables_x = [
    SparsePauliOp(id[:i] + "X" + id[(i + 1) :]).apply_layout(
        transpiled_circuit.layout
    )
    for i in range(num_qubits)
]
observables_y = [
    SparsePauliOp(id[:i] + "Y" + id[(i + 1) :]).apply_layout(
        transpiled_circuit.layout
    )
    for i in range(num_qubits)
]
observables_z = [
    SparsePauliOp(id[:i] + "Z" + id[(i + 1) :]).apply_layout(
        transpiled_circuit.layout
    )
    for i in range(num_qubits)
]

# We define the primitive unified blocs (PUBs) consisting of the embedding circuit,
# set of observables and the circuit parameters
pub_x = (transpiled_circuit, observables_x)
pub_y = (transpiled_circuit, observables_y)
pub_z = (transpiled_circuit, observables_z)

# Experiment options for error mitigation
num_randomizations = 300
shots_per_randomization = 100
noise_factors = [1, 3, 5]

experimental_opts = {}
experimental_opts["resilience"] = {
    "measure_mitigation": True,
    "zne_mitigation": True,
    "zne": {
        "noise_factors": noise_factors,
        "amplifier": "gate_folding",
        "extrapolated_noise_factors": [0] + noise_factors,
    },
}
experimental_opts["twirling"] = {
    "num_randomizations": num_randomizations,
    "shots_per_randomization": shots_per_randomization,
    "strategy": "active-accum",
}

# We define and run the estimator to obtain <X>, <Y> and <Z> on all qubits
estimator = Estimator(mode=backend, options=experimental_opts)

job = estimator.run([pub_x, pub_y, pub_z])

Tukuyin ang projected quantum kernel​

Ang projected quantum kernel ay tinukoy gamit ang sumusunod na kernel function: kPQ(xi,xj)=exp(−γ∑k∑P∈{X,Y,Z}(Tr[Pρk(xi)]−Tr[Pρk(xj)])2)k^{\textrm{PQ}}(x_i, x_j) = \textrm{exp} \Big(-\gamma \sum_k \sum_{P \in \{ X,Y,Z \}} (\textrm{Tr}[P \rho_k(x_i)] - \textrm{Tr}[P \rho_k(x_j)])^2 \Big) kPQ(xi​,xj​)=exp(−γ∑k​∑P∈{X,Y,Z}​(Tr[Pρk​(xi​)]−Tr[Pρk​(xj​)])2) Sa equation sa itaas, ang γ>0\gamma>0γ>0 ay isang naaayos na hyperparameter. Ang KijPQ=kPQ(xi,xj)K^{\textrm{PQ}}_{ij} = k^{\textrm{PQ}}(x_i, x_j)KijPQ​=kPQ(xi​,xj​) ay mga entry ng kernel matrix na KPQK^{\textrm{PQ}}KPQ. Gamit ang kahulugan ng 1-RDMs, makikita natin na ang bawat term sa loob ng kernel function ay maaaring suriin bilang Tr[Pρk(xi)]=⟨P⟩\textrm{Tr}[P \rho_k (x_i)] = \braket PTr[Pρk​(xi​)]=⟨P⟩, kung saan ang P∈{X,Y,Z}P \in \{ X,Y,Z \}P∈{X,Y,Z}. Ang mga expectation value na ito ay eksakto ang mga nasukat natin sa itaas. Sa pamamagitan ng scikit-learn, maaari nating kalkulahin ang kernel nang mas madali pa. Ito ay dahil sa madaling magamit na radial basis function ('rbf') kernel: exp(−γ∥x−x′∥2) \textrm{exp} (-\gamma \lVert x - x' \rVert^2)exp(−γ∥x−x′∥2). Una, kailangan lamang nating baguhin ang hugis ng bagong projected na mga training at test dataset sa dalawang dimensyonal na array. Tandaan po na ang pagdaan sa buong dataset ay maaaring tumagal ng humigit-kumulang 80 minuto sa QPU. Upang masiguro na ang natitirang bahagi ng tutorial ay madaling maipatupad, nagbibigay din kami ng mga projection mula sa isang nakaraang eksperimento (na kasama sa mga file na na-download ninyo sa Download dataset code block). Kung ikaw ay nagsagawa ng pagsasanay sa iyong sarili, maaari kang magpatuloy sa tutorial gamit ang iyong sariling mga resulta.

Apendiks: I-verify ang potensyal na quantum advantage ng dataset sa mga gawain ng pag-aaral​

Hindi lahat ng dataset ay nag-aalok ng potensyal na kalamangan mula sa paggamit ng PQKs. Mayroong ilang teoretikal na hangganan na maaaring gamitin bilang paunang pagsubok upang makita kung ang isang partikular na dataset ay makikinabang mula sa PQKs. Upang masukat ito, tinukoy ng mga may-akda ng Power of data in quantum machine learning [2] ang mga dami na tinutukoy bilang klasikal at quantum na model complexities at geometric separation ng mga klasikal at quantum na modelo. Upang asahan ang potensyal na quantum advantage mula sa PQKs, ang geometric separation sa pagitan ng mga klasikal at quantum-projected na kernel ay dapat na humigit-kumulang sa order ng N\sqrt{N}N​, kung saan ang NNN ay ang bilang ng mga training sample. Kung natutugunan ang kondisyong ito, magpapatuloy tayo sa pagsuri ng mga model complexity. Kung ang klasikal na model complexity ay nasa order ng NNN habang ang quantum-projected na model complexity ay mas maliit nang malaki kaysa sa NNN, maaari tayong umasa ng potensyal na kalamangan mula sa PQK. Ang geometric separation ay tinukoy tulad ng sumusunod (F19 sa [2]): gcq=g(Kc∥Kq)=∥KqKc(Kc+λI)−2KcKq∥∞g_{cq} = g(K^c \Vert K^q) = \sqrt{\Vert \sqrt{K^q} \sqrt{K^c} (K^c + \lambda I)^{-2} \sqrt{K^c} \sqrt{K^q}\Vert_{\infty}}gcq​=g(Kc∥Kq)=∥Kq​Kc​(Kc+λI)−2Kc​Kq​∥∞​​

# Gamma values used in best models above
