Shor's algorithm

Hindi pa naisalin

Ang pahinang ito ay hindi pa naisalin. Nakikita mo ang orihinal na bersyon sa Ingles.

Now we'll turn our attention to the integer factorization problem, and see how it can be solved efficiently on a quantum computer using phase estimation. The algorithm we'll obtain is Shor's algorithm for integer factorization. Shor didn't describe his algorithm specifically in terms of phase estimation, but it is a natural and intuitive way to explain how it works.

We'll begin by discussing an intermediate problem known as the order-finding problem and see how phase estimation provides a solution to this problem. We'll then see how an efficient solution to the order-finding problem gives us an efficient solution to the integer factorization problem. (When a solution to one problem provides a solution to another problem like this, we say that the second problem reduces to the first — so in this case we're reducing integer factorization to order finding.) This second part of Shor's algorithm doesn't make use of quantum computing at all; it's completely classical. Quantum computing is only needed to solve order finding.

The order-finding problem​

Some basic number theory​

To explain the order-finding problem and how it can be solved using phase estimation, it will be helpful to begin with a couple of basic number theory concepts, and to introduce some handy notation along the way.

To begin, for any given positive integer N,N,N, define the set ZN\mathbb{Z}_NZN​ like this.

ZN={0,1,…,N−1}\mathbb{Z}_N = \{0,1,\ldots,N-1\}ZN​={0,1,…,N−1}

For instance, Z1={0},  \mathbb{Z}_1 = \{0\},\;Z1​={0}, Z2={0,1},  \mathbb{Z}_2 = \{0,1\},\;Z2​={0,1}, Z3={0,1,2},  \mathbb{Z}_3 = \{0,1,2\},\;Z3​={0,1,2}, and so on.

These are sets of numbers, but we can think of them as more than sets. In particular, we can think about arithmetic operations on ZN\mathbb{Z}_NZN​ such as addition and multiplication — and if we agree to always take our answers modulo NNN (that is, divide by NNN and take the remainder as the result), we'll always stay within this set when we perform these operations. The two specific operations of addition and multiplication, both taken modulo N,N,N, turn ZN\mathbb{Z}_NZN​ into a ring, which is a fundamentally important type of object in algebra.

For example, 333 and 555 are elements of Z7,\mathbb{Z}_7,Z7​, and if we multiply them together we get 3⋅5=15,3\cdot 5 = 15,3⋅5=15, which leaves a remainder of 111 when divided by 7.7.7. Sometimes we express this as follows.

Problem statement and connection to phase estimation​

Now we can state the order-finding problem.

Order finding

Order finding through phase estimation​

To solve the order-finding problem for a given choice of a∈ZN∗,a\in\mathbb{Z}_N^{\ast},a∈ZN∗​, we can apply the phase-estimation procedure to the operation Ma.M_a.Ma​.

To do this, we need to implement not only MaM_aMa​ efficiently with a quantum circuit, but also Ma2,M_a^2,Ma2​, Ma4,M_a^4,Ma4​, Ma8,M_a^8,Ma8​, and so on, going as far as needed to obtain a precise enough estimate from the phase estimation procedure. Here we'll explain how this can be done, and we'll figure out exactly how much precision is needed later.

Let's start with the operation MaM_aMa​ by itself. Naturally, because we're working with the quantum circuit model, we'll use binary notation to encode the numbers between 000 and N−1.N-1.N−1. The largest number we need to encode is N−1,N-1,N−1, so the number of bits we need is

n=lg⁡(N−1)=⌊log⁡(N−1)⌋+1.n = \operatorname{lg}(N-1) = \lfloor \log(N-1) \rfloor + 1.n=lg(N−1)=⌊log(N−1)⌋+1.

For example, if N=21N = 21N=21 we have n=lg⁡(N−1)=5.n = \operatorname{lg}(N-1) = 5.n=lg(N−1)=5. Here's what the encoding of elements of Z21\mathbb{Z}_{21}Z21​ as binary strings of length 555 looks like.

Ma∣x⟩={∣ax  (mod  N)⟩0≤x<N∣x⟩N≤x<2nM_a \vert x\rangle = \begin{cases} \vert ax \; (\textrm{mod}\;N)\rangle & 0\leq x < N\\[1mm] \vert x\rangle & N\leq x < 2^n \end{cases}Ma​∣x⟩={∣ax(modN)⟩∣x⟩​0≤x<NN≤x<2n​

To perform Ma2,M_a^2,Ma2​, Ma4,M_a^4,Ma4​, Ma8,M_a^8,Ma8​, and so on, we can use exactly the same method, except that we replace aaa with a2,a^2,a2, a4,a^4,a4, a8,a^8,a8, and so on, as elements of ZN∗.\mathbb{Z}_N^{\ast}.ZN∗​. General solution​

As we just saw, if we have the eigenvector ∣ψ1⟩\vert \psi_1 \rangle∣ψ1​⟩ of Ma,M_a,Ma​, we can learn rrr through phase estimation, so long as we use enough control qubits to do this with sufficient precision. Unfortunately, it's not easy to get our hands on the eigenvector ∣ψ1⟩,\vert\psi_1\rangle,∣ψ1​⟩, so we need to figure out how to proceed.

Let's suppose momentarily that we proceed just like above, except with the eigenvector ∣ψk⟩\vert\psi_k\rangle∣ψk​⟩ in place of ∣ψ1⟩,\vert\psi_1\rangle,∣ψ1​⟩, for any choice of k∈{0,…,r−1}k\in\{0,\ldots,r-1\}k∈{0,…,r−1} that we choose to think about. The result we get from the phase estimation procedure will be an approximation

y2m≈kr.\frac{y}{2^m} \approx \frac{k}{r}.2my​≈rk​.

Working under the assumption that we don't know either kkk or r,r,r, this might or might not allow us to identify r.r.r. For example, if k=0k = 0k=0 we'll get an approximation y/2my/2^my/2m to 0,0,0, which unfortunately tells us nothing. This, however, is an unusual case; for other values of k,k,k, we'll at least be able to learn something about r.r.r.

We can use an algorithm known as the continued fraction algorithm to turn our approximation y/2my/2^my/2m into nearby fractions — including k/rk/rk/r if the approximation is good enough. We won't explain the continued fraction algorithm here. Instead, here's a statement of a known fact about this algorithm.

Fact

This algorithm can be implemented as a Boolean circuit having size O((lg⁡(N))3).O((\operatorname{lg}(N))^3).O((lg(N))3).

If we have a very close approximation y/2my/2^my/2m to k/r,k/r,k/r, and we run the continued fraction algorithm for NNN and α=y/2m,\alpha = y/2^m,α=y/2m, we'll get uuu and v,v,v, as they're described in the fact. An analysis of the fact allows us to conclude that

uv=kr.\frac{u}{v} = \frac{k}{r}.vu​=rk​.

Notice in particular that we don't necessarily learn kkk and r,r,r, we only learn k/rk/rk/r in lowest terms.

For example, and as we've already noticed, we're not going to learn anything from k=0.k=0.k=0. But that's the only value of kkk where that happens. When kkk is nonzero, it might have common factors with r,r,r, but the number vvv we obtain from the continued fraction algorithm must at least divide r.r.r.

It's far from obvious, but it is true that if we have the ability to learn uuu and vvv for u/v=k/ru/v = k/ru/v=k/r for k∈{0,…,r−1}k\in\{0,\ldots,r-1\}k∈{0,…,r−1} chosen uniformly at random, then we're very likely to be able to recover rrr after just a few samples. But random choices of kkk aren't likely to hide factors of rrr for long, and the probability that we don't guess rrr correctly by taking the least common multiple of the denominators we observe drops exponentially in the number of samples.

Total cost​

The cost to implement each controlled-unitary MakM_a^kMak​ is O(n2).O(n^2).O(n2). In addition, we have mmm Hadamard gates (which contribute O(n)O(n)O(n) to the cost), and the inverse quantum Fourier transform contributes O(n2)O(n^2)O(n2) to the cost. In addition to the quantum circuit itself, there are a few classical computations that need to be performed along the way. Factoring by order finding​

The very last thing we need to discuss is how solving the order-finding problem helps us to factor. This part is completely classical — it has nothing specifically to do with quantum computing.

Here's the basic idea. We want to factorize the number N,N,N, and we can do this recursively. Specifically, we can focus on the task of splitting N,N,N, which means finding any two integers b,c≥2b,c\geq 2b,c≥2 for which N=bc.N = bc.N=bc. This isn't possible if NNN is a prime number, but we can efficiently test to see if NNN is prime using a primality testing algorithm first, and if NNN isn't prime we'll try to split it. Once we split N,N,N, we can simply recurse on bbb and ccc until all of our factors are prime and we obtain the prime factorization of N.N.N.

Splitting even integers is easy: we just output 222 and N/2.N/2.N/2.

It's also easy to split perfect powers, meaning numbers of the form N=sjN = s^jN=sj for integers s,j≥2,s,j\geq 2,s,j≥2, just by approximating the roots N1/2,N^{1/2},N1/2, N1/3,N^{1/3},N1/3, N1/4,N^{1/4},N1/4, and so on, and checking nearby integers as suspects for s.s.s. We don't need to go further than log⁡(N)\log(N)log(N) steps into this sequence, because at that point the root drops below 222 and won't reveal additional candidates.

It's good that we can do both of these things because order finding won't help us to factor even numbers or for prime powers, where the number sss happens to be prime. If NNN is odd and not a prime power, however, order finding allows us to split N.N.N.

Probabilistic algorithm to split an odd, composite integer N that is not a prime power

  1. Randomly choose a∈{2,…,N−1}.a\in\{2,\ldots,N-1\}.a∈{2,…,N−1}.

  2. Compute d=gcd⁡(a,N).d=\gcd(a,N).d=gcd(a,N).

  3. If d>1d > 1d>1 then output b=db = db=d and c=N/dc = N/dc=N/d and stop. Otherwise continue to the next step knowing that a∈ZN∗.a\in\mathbb{Z}_N^{\ast}.a∈ZN∗​.

  4. Let rrr be the order of aaa modulo N.N.N. (Here's where we need order finding.)

  5. If rrr is even:

    5.1 Compute x=ar/2−1x = a^{r/2} - 1x=ar/2−1 modulo NNN
    5.2 Compute d=gcd⁡(x,N).d = \gcd(x,N).d=gcd(x,N).
    5.3 If d>1d>1d>1 then output b=db=db=d and c=N/dc = N/dc=N/d and stop.

  6. If this point is reached, the algorithm has failed to find a factor of N.N.N.

A run of this algorithm may fail to find a factor of N.N.N. Specifically, this happens in two situations:

  • The order of aaa modulo NNN is odd.
  • The order of aaa modulo NNN is even and gcd⁡(ar/2−1,N)=1.\gcd\bigl(a^{r/2} - 1, N\bigr) = 1.gcd(ar/2−1,N)=1.

Using basic number theory it can be proved that, for a random choice of a,a,a, with probability at least 1/21/21/2 neither of these events happen. In fact, the probability that either event happens is at most 2−(m−1)2^{-(m-1)}2−(m−1) for mmm being the number of distinct prime factors of N,N,N, which is why the assumption that NNN is not a prime power is needed. (The assumption that NNN is odd is also required for this fact to be true.)

This means that each run has at least a 50% chance to split N.N.N. Therefore, if we run the algorithm ttt times, randomly choosing aaa each time, we'll succeed in splitting NNN with probability at least 1−2−t.1 - 2^{-t}.1−2−t.

The basic idea behind the algorithm is as follows. If we have a choice of aaa for which the order rrr of aaa modulo NNN is even, then r/2r/2r/2 is an integer and we can consider the numbers

ar/2−1  (mod  N)andar/2+1  (mod  N).a^{r/2} - 1\; (\textrm{mod}\; N) \quad \text{and} \quad a^{r/2} + 1\; (\textrm{mod}\; N).ar/2−1(modN)andar/2+1(modN).

Using the formula Z2−1=(Z+1)(Z−1),Z^2 - 1 = (Z+1)(Z-1),Z2−1=(Z+1)(Z−1), we conclude that

(ar/2−1)(ar/2+1)=ar−1.\bigl(a^{r/2} - 1\bigr) \bigl(a^{r/2} + 1\bigr) = a^r - 1.(ar/2−1)(ar/2+1)=ar−1.

Now, we know that ar  (mod  N)=1a^r \; (\textrm{mod}\; N) = 1ar(modN)=1 by the definition of the order — which is another way of saying that NNN evenly divides ar−1.a^r - 1.ar−1. That means that NNN evenly divides the product

(ar/2−1)(ar/2+1).\bigl(a^{r/2} - 1\bigr) \bigl(a^{r/2} + 1\bigr).(ar/2−1)(ar/2+1).

For this to be true, all of the prime factors of NNN must also be prime factors of ar/2−1a^{r/2} - 1ar/2−1 or ar/2+1a^{r/2} + 1ar/2+1 (or both) — and for a random selection of aaa it turns out to be unlikely that all of the prime factors of NNN will divide one of the terms and none will divide the other. Otherwise, so long as some of the prime factors of NNN divide the first term and some divide the second term, we'll be able to find a non-trivial factor of NNN by computing the GCD with the first term.
